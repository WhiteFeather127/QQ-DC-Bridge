# QQ-DC-Bridge 使用说明

## 概述

QQ-DC-Bridge 是一个跨平台消息桥接机器人，在 **QQ 群**（通过 OneBot/NapCat 协议）和 **Discord 频道**（通过 Discord Bot API）之间进行双向实时消息转发，并附带自动翻译功能。

---

## 1. 支持的消息类型

| 类型 | 说明 | 跨平台转发表现 |
|---|---|---|
| **文字** | 纯文本消息 | 正常转发，并经过自动翻译 |
| **图片** | 单张或多张图片 | QQ ↔ Discord 双向转发 |
| **@提及** | @群成员 / @特定用户 | 转发时自动转换为对端平台的 @格式（需已绑定账号） |
| **@全体成员** | @everyone | 降级为文字提示转发 |
| **回复/引用** | 回复某条消息 | 保留引用关系，转发为对端平台的回复格式 |
| **Emoji** | QQ 原生表情 / Unicode Emoji | QQ 表情 ID 自动映射为标准 Unicode Emoji |
| **Discord 贴纸** | Discord Sticker | 自动转为图片转发 |
| **不支持的类型** | 无法识别的消息段 | 降级为占位文字提示 |

> **注意**：转发图片时，QQ 侧依赖本地文件缓存或 CDN 直链，部分场景下可能需要 NapCat 配置正确的图片访问权限。

---

## 2. 账号绑定 (`/bind`)

绑定让 QQ 账号和 Discord 账号建立一一对应关系，主要用于：
- **显示名替换**：转发消息时显示对端平台的昵称
- **跨平台 @ 解析**：@某人时自动转换为对端平台可识别的格式
- **文本 @ 识别**：在消息中写 `@对方昵称`，若匹配绑定的用户，自动转为对端平台的 @

### 绑定流程

绑定通过 Bot 私信完成，**目前仅支持从 Discord 侧发起**：

1. 在 Discord 向 Bot 私信发送 `/bind <QQ号>`（例如 `/bind 123456789`）
2. Bot 检查双方均未被绑定后，生成 6 位数字验证码
3. 验证码通过 **QQ 侧私信** 发送给目标 QQ 用户
4. 目标 QQ 用户将该 6 位验证码回复给 Bot 私信
5. Bot 校验通过，绑定完成

### 其他绑定相关命令

| 命令 | 说明 | 可用平台 |
|---|---|---|
| `/bind <QQ号>` | 发起绑定流程 | Discord |
| `/unbind` | 解除当前账号绑定 | Discord / QQ |

### 注意事项

- 验证码有效期 **5 分钟**
- 验证码错误尝试上限 **5 次**，超限后需重新发起绑定
- QQ 账号和 Discord 账号均为**一对一绑定**，任一端已绑定时无法再次绑定

---

## 3. 触发方式

### QQ 侧：需 @Bot 或回复 Bot 消息

QQ 群中的消息**不会全部转发**，只有满足以下条件之一的消息才会被桥接到 Discord：

- **消息中 @了 Bot**（即消息包含 @机器人）
- **消息是回复**了一条由 Bot 发出的消息

不满足以上条件的 QQ 群消息会被忽略。

> **原因**：QQ 群消息量大，如果全部转发会产生大量噪音。@Bot 或回复 Bot 是用户主动与 Bot 交互的明确信号。

### Discord 侧：自动触发

Discord 频道中的**所有消息**都会被自动转发到 QQ 群（除非消息由 Bot 自身发出）。

> **原因**：Discord 频道通常是桥接的发起端，消息量相对可控，因此采用全量转发策略。

### 断线重连补发（仅 Discord）

Discord 侧支持断线重连后的消息补发（Catch-up）：
- 自动记录最后处理的消息 ID
- 重连后拉取断开期间漏掉的消息
- 最多补发 **30 分钟**内的消息
- QQ 侧不支持补发，断开期间的消息会丢失

---

## 4. 自动翻译

- 翻译引擎：**DeepSeek API**（兼容 OpenAI 格式）
- 翻译方向：
  - QQ → Discord：译为**英文**
  - Discord → QQ：译为**中文**
- 纯链接/图片消息**跳过翻译**
- 译文与原文相同时自动跳过（避免无效翻译）
- 翻译失败时**降级为原文转发**，不影响消息投递

### 翻译控制

在消息开头添加 `/distrans` 即可**跳过翻译**，直接发送原文：

```
/distrans 这是一条中文消息，请不要翻译
```

> 注意：`/distrans` 必须写在消息最开头，且仅对当前消息生效。

---

## 5. 命令汇总

| 命令 | 说明 | 发送位置 | 可用平台 |
|---|---|---|---|
| `/bind <QQ号>` | 发起账号绑定 | Bot 私信 | Discord |
| `/unbind` | 解除账号绑定 | Bot 私信 | Discord / QQ |
| `<6位验证码>` | 完成绑定第二步 | Bot 私信 | QQ |
| `/distrans <消息>` | 跳过翻译，发送原文 | 群聊/频道 | Discord / QQ |

---

## 6. 已知限制

- **QQ 侧不支持消息编辑转发**：OneBot 协议无编辑事件通知
- **QQ 侧不支持断线补发**：断开连接期间的消息会丢失
- **绑定仅支持从 Discord 发起**：QQ 侧发起的绑定流程已被禁用
- **图片转发可能受限**：取决于 NapCat 的图片访问配置
- **速率限制**：QQ 侧默认限速 1 条/秒（突发 3 条），超限消息会被丢弃
